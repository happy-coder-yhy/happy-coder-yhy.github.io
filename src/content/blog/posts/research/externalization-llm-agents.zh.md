## 论文信息

- **标题**：Externalization in LLM Agents: A Unified Review of Memory, Skills, Protocols and Harness Engineering
- **作者**：Zhou 等
- **年份**：2026
- **类型**：综述（Review）

## 核心框架：四维知识外化

本文提出了 LLM Agent 中知识外化的统一框架，包含四个核心维度：

### 1. 记忆系统（Memory）

Agent 的记忆管理是知识外化的基础，主要包括：
- **短期记忆**：上下文窗口内的信息维护
- **长期记忆**：跨会话的知识持久化与检索
- **工作记忆**：当前任务执行过程中的临时状态

### 2. 技能编排（Skills）

Agent 如何组织和调用其能力：
- **工具使用（Tool Use）**：调用外部 API 和工具
- **技能组合（Skill Composition）**：将多个技能串联为复杂工作流
- **能力发现（Capability Discovery）**：动态识别和获取新技能

### 3. 通信协议（Protocols）

Agent 内部及 Agent 间的通信机制：
- **MCP（Model Context Protocol）**：标准化的工具接入协议
- **Agent 间通信**：多 Agent 协作的消息传递
- **人机交互**：Agent 与用户的交互接口

### 4. Harness 工程

Agent 系统的工程化基础设施：
- **实验平台**：统一的实验管理与追踪
- **评估框架**：多维度 Agent 性能评测
- **部署运维**：Agent 系统的生产化部署

## 个人思考

### 1. 论文想解决什么问题？

#### 核心问题
当前LLM Agent领域的进展虽然迅速，但呈现出碎片化状态（如记忆系统、工具使用、多智能体协作等各自发展），缺乏一个统一的理论逻辑来解释这些不同方向为何会同时涌现，以及它们之间如何关联。

#### 具体目标
- 提出“外部化”（Externalization）作为贯穿LLM Agent设计的主线逻辑，解释为何将认知负担从模型内部移出（到记忆、技能、协议等外部结构）能带来更可靠、可治理的智能体行为。
- 构建一个系统级框架，将记忆（Memory）、技能（Skills）、协议（Protocols）和控制框架（Harness）统一纳入同一分析视角，阐明它们的耦合关系与分工。
- 引导未来研究从“只关注模型本身”转向“模型与外部基础设施协同演化”的系统观。

### 2. 前人已有哪些工作，痛点是什么？

#### 前人工作
- **基于权重的能力（Weights Era）**：如GPT-4、DeepSeek等，通过大规模预训练和微调将知识、推理能力压缩在模型参数中。
- **基于上下文的能力（Context Era）**：如Prompt Engineering、Chain-of-Thought、ReAct、RAG等，通过动态注入示例、推理步骤或检索文档来引导模型行为。
- **基础设施级的能力（Harness Era）**：如AutoGPT、BabyAGI、LangGraph、AutoGen等，引入外部循环、任务队列、多智能体协作等结构。

#### 痛点
- **碎片化**：记忆、工具、多智能体协作等研究彼此隔离，缺乏统一解释。
- **可靠性瓶颈**：模型在长时任务、多步推理、工具调用、跨会话交互中表现不稳定，容易出现遗忘、步骤跳过、格式错误等问题。
- **治理困难**：模型行为难以审计、约束、回滚，尤其在安全敏感场景中。
- **评估偏误**：现有基准主要测试模型本身能力，忽略了外部基础设施对性能的巨大贡献，导致“模型进步”被高估。

### 3. 本文用了什么创新方法解决了前人的痛点？

#### 创新方法
提出并系统化 **“外部化”理论框架**，将智能体设计重新定义为“将认知负担从模型内移到外部结构”的过程。具体包括：

- **三个外部化维度**：
  - **记忆（Memory）**：外部化跨时间的状态（如工作上下文、情景经验、语义知识、个性化信息），将“回忆”问题转化为“识别+检索”问题。
  - **技能（Skills）**：外部化程序性专家知识（操作流程、决策启发式、规范约束），将“即兴生成”转化为“选择+组合”问题。
  - **协议（Protocols）**：外部化交互结构（工具调用、智能体间协作、用户交互），将“临时推断”转化为“结构化交换”问题。

- **Harness（控制框架）**：作为统一层，协调上述三个模块，提供循环控制、沙盒隔离、人工审批、可观测性、策略编码、上下文预算管理等运行机制。

- **理论支撑**：引入认知科学中的**认知工具（Cognitive Artifacts）** 和**互补策略（Complementary Strategies）** 概念，强调外部结构通过“任务重组”而非“简单增强”提升智能体可靠性。

通过这一框架，论文将分散的研究成果映射到统一的“外部化”逻辑下，解释为何这些进展能提升可靠性、可治理性和可复用性。

### 4. 效果怎么样？

作为一篇综述论文，其“效果”体现在以下方面：

- **统一了领域认知**：为记忆、技能、协议、控制框架等子领域提供了共同的逻辑基础，使研究者能更清晰地定位各自工作在整体框架中的位置。
- **解释了实践趋势**：解释了为什么现代Agent系统（如Claude Code、OpenAI Codex、AutoGen等）越来越多地将能力“外移”到外部结构，而非仅依赖模型升级。
- **指出了设计原则**：提出了六维度Harness设计框架（循环控制、沙盒、审批、可观测性、配置、上下文管理），为工程实践提供了系统化指导。
- **为未来研究指明方向**：包括多模态外部化、具身智能体、自演化Harness、共享基础设施、安全性治理、评估方法等。

虽然没有定量实验结果，但论文通过大量现有系统案例（如MemGPT、Voyager、MCP、A2A等）验证了其框架的解释力和覆盖面。

### 5. 还有什么不足之处（未来继续深挖的点有哪些）？

论文在“未来讨论”（Section 8）中明确指出了多个开放问题和不足：

- **边界动态性**：模型能力增强可能“内化”部分外部功能，如何动态调整外部化边界仍需研究。
- **多模态与具身化**：当前框架以文本为主，如何扩展到视觉、语音、物理动作等模态，以及具身智能体中的“大脑-小脑”分工仍需深入。
- **自演化Harness**：如何让Harness自身具备自适应、自优化能力（如通过RL、程序合成、演化搜索）仍处于早期探索阶段。
- **成本与风险**：
  - **认知开销**：外部化过度可能导致模型花费大量精力在模块间协调，而非任务本身。
  - **安全威胁**：如记忆投毒、恶意技能注入、协议欺骗等，外部化扩大了攻击面。
- **共享基础设施**：从私有脚手架走向共享记忆/技能/协议时，如何治理版本、权限、质量、信任等问题仍不成熟。
- **评估缺失**：现有基准未能有效度量外部化带来的增益（如迁移性、维护性、恢复鲁棒性、上下文效率、治理质量），亟需新评估框架。

此外，论文整体偏理论综合，缺乏量化实验或系统比较，未来可在此基础上开展实证研究，验证“外部化程度”与“任务成功率、成本、安全性”之间的量化关系。
